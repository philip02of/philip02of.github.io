Over Mij

Beste bezoeker van de website,


Ik ben Philip Offermans en ik ben 16 jaar oud.

Momenteel zit ik op 4VWO van het Grotius College. Vorig schooljaar moest iedereen een keuzevak kiezen, dit keuze vak moet je naast je regulieren lessen volgen. Ik had gekozen voor informatica, dit omdat ik het heel interessantvind en er naar mijn mening altijd nieuwe dingen zijn te leren. Deze website is gemaakt voor informatica.


Heb ik er iets van geleerd?

Ja, zo heb ik geleerd hoe je animaties met css kan maken (mischien leuk voor in een les), ook heb ik voor het eerst een website zonder bootstrap gemaakt. Ik heb alles zelf geprogameerd behalve de shake-animaties. Ik heb wel uitgezocht hoe de maker van de shake-anitmaties deze gemaakt heeft. Het is vrij simpel en ik snap de code.

Begrippen

Fysiekelaag: Hardware waaruit een apperaat bestaat.
Logischelaag: Software die de hardware aanstuurt.
Toepassingenlaag: Software die je kunt gebruiken.
Event Handler: Code die ervoor zorgt dat er iets uitgevoert wordt als er iets bepaalds gebeurt.
Interfaces: Verbindingen tussen verschilende lagen.
Hardware-interfaces: Fysieke verbindingen. Bijvoorbeeld: op een moederbord.
Printplaat: Plaatje die contactpunten met elkaar verbind.
Chips: Aparaatjes op een plaatje.
Connectoren: Verbindingen die je makkelijk los en vast kan maken. Bijvoorbeeld: USB
Drivers: Stuurt hardware aan.
API’s: Software liberies/pakketen die ervoor zorgen dat programma’s makkelijk met elkaar comuniceren.
Consistentie: Gebruikers krijgen steets iets te zien dat herkenbaar is.
Web-API: Een liberie die het makkelijk maakt data op te halen (volgens boek).
Userinterface: Via deze interface comuniceert een apperaat met de gebruiker.
GUI: Dit is een grafische userinterface.
Events: Dingen die een computer kan waarnemen waarop een actie uitgevoerd kan worden. Bijvoorbeeld: muisbeweging maar dit kan ook een weersverandering zijn die een sensor meet.
VUI: Een userinterface die met de stem bediend word. Bijvoorbeeld: Google Home.
CLI: Hiermee kun je via comando’s de computer besturen.
Remote: Een server op afstand besturen. Bij linux gebeurt dit meestal via SSH (Secure Shell).
Lan: Lokaal Netwerk (meestal via een modem aangesloten op de buitenwereld).
WIFI: Draadloze verbinding met een netwerk. Dit gaat via stralingen.
4G: een versie van het Mobiele netwerk (1g 2g 3g 4g en ook al 5g slecht benoemd)
Netwerkprotocollen: Dit zijn afspraken hoe aparaten met elkaar praten via een netwerk. Er zijn veel verschillende standaarden.
HTTP: dit protocol gaat over hoe websites via internet bezocht kunnen worden.
Clienst: Dit zijn aparaten die gebruik maken van een service.
Server: Dit zijn aparaten die een service verlenen.
Parellelle verwerking: Meerdere bewerking op een zelfde moment.
Raspberry Pi: Super vet klein computertje. Het draait makkelijk linux en je kan er coolen projecten mee maken.
PC: Computers geschikt voor huis, tuin (niet aan te raden) en keuken gebruik.
Mac: Een computer van Apple.
Embedded system: Systeem waarbij de software en hardware volledig voor elkaar zijn gemaakt. Bijvoorbeeld: Google Home.
Onopvallenheid: De gebruiker heeft niet door dat er een computer in bijvoorbeeld een product verwerkt zit.
Betrouwbaarheid: Embeded systems moeten lang blijven werken omdat ze meestal moeilijkt te vervangen zijn.
Zuinigheid: Een embeded system moet weinig energie verbruik hebben omdat deze meestal 24/7 aanstaat.
Beperktheid: Embeded systems zijn meestal gemaakt voor 1 doel. Je kan er dus geen andere software op draaien.
Iot: Normale producten die door computers slim gemaakt worden.
Kernaparatuur: Hardware die elke computer heeft.
Randapparatuur: Hardware die aangesloten kan worden op een computer.
Moederbord: Een printplaat van een computer die alle kernapparatuur met elkaar verbind.
System on a chip: Één chip met meerdere functies die aangesloten zit op het moederbord. Bijvoorbeeld: een cpu met een ingebouwde videokaart.
Multi-processorsysteem: Een computer met meerdere procesoren. Bijvoorbeeld: een rekencomputer in een data center.
Multicoreprocessor: Een procesor met meerdere kernen, dit zorgt ervoor dat de computer meerde taken teglijk kan uitvoeren.
Pipelining: Opdrachten die de computer moet uitvoeren worden verdeeeld over de meeerdere cores.
Processor: Berekent alle opdrachten voor de computer (werkt alleen als hij opdrachten krijgt).
Read Only Memory: Geheugen dat alleen gelezen kan worden, er kan dus niet in worden geschreven.
EEPROM: Opslagvorm waarmee alles met electrische spanningen wordt opgeslagen.
BIOS: De gegevens die de computer nodig heeft om met de belangrijkste randapparaten (extern geheugen, toetsenbord en dergelijke) te kunnen werken.
POST: Programma dat een aantal onderdelen van de computer test.
Setup: GUI van het bios. Je kan hierin veranderingen aanbrengen.
Random Access Memory: Werkgeheugen is een vast onderdeel in een computer. Het is een tijdelijk geheugen wat de processor gebruikt om informatie dicht en op een snelle manier bij zich te houden.
Sensoren: De zintuigen van een aparaat. Bijvoorbeeld: een camerasensor om beeld waar te nemen.
Analoog signaal: Een singniaal dat niet digitaal in computer zit.
Versnellingsmeter: Sensor die versnelling meet.
Sensoren: In het boek staan veel sensoren aangegeven. Het spreekt voor zich. De sensoren nemen dingen waar. Elke sensor neemt iets anders waar.
Actuatoren: Hardware die een output geeft die de gebruiker weer kan waarnemen.
Communicatiechips: Dit zijn chips die de mogelijk hebben om te comuniceren door signiaalen uit te zenden en te ontvangen.
RJ45-connector: Standaard connector voor bekaabelde netwerken (niet de enige je hebt bijv. ook een glasvezel aansluiting).
flashgeheugen: Schijf waarop je kunt lezen en schrijven.
Solid State Disk: Een geheugenschijf bestaand uit dezelfde chips als in een sd-kaart.
Emulator: Een programma die zich voor doet als bepaalde hardware
Virtual machines: Een os dat draait in een virtuele omgeving. Een soort virtuele doos. Daarom heet het meest gebruikte Virtual machine host programme "VirtualBox"
Hosten: Huren van een computer die ergens anders staat.
